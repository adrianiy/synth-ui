# Styles Provider
##### StylesProvider

A provider to allow use of css variables across your application.

You can also set the prefered value for `theme`, `interface` and `basePath`. These values will be used as default property values in every `glyph-component`.

> If you are using Amiga-fwk you should use this provider because css-next plugin makes glyph variables break.

> Changing theme value will recalculta global css variables values.

This provider will allow you to get `glyphTheme` and `glyphInterface` values anywhere using `useGlyphStyle`.

```typescript
// main.tsx
import { GlyphStyleProvider } from 'glyph-react-poc';

...
const home = () => {
    const glyphInterface = localStorage.getItem("drive.warehouses.interface") || UIInterface.redesign;
    const glyphTheme = localStorage.getItem("drive.warehouses.theme") || UITheme.light;

    return (
        ...
        <GlyphStyleProvider interface={glyphInterface} theme={glyphTheme} basePath="/drfrwrhs/web">
            ...
        </GlyphStyleProvider>
        ...
    )
}
```

## Theme

```typescript
const { glyphTheme, setGlyphTheme } = useGlyphStyle();
```

GlyphUI implements two theme `dark` and `light`. Using this provider you can change it or use current values
wherever you want

> Default theme is `light`

## Interface

```typescript
const { glyphInterface, setGlyphInterface } = useGlyphStyle();
```

Currently we have `classic`, `modern` and a brand new `redesign` interface we are working in.

As soon as we can `redesign` interface will become the default one but in the meantime you can set
this interface manually in glyph-react components.

Web Components are shadowed and independent so each component must receive an interface value.

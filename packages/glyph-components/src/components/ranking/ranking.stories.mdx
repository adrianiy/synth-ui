# Ranking

##### glyph-ranking

This component can be rendered as a single ranking (grid) or a ranking with three columns (woman, man, kid).

The **articles** can have a propert `image` or `plain_image` with a full url, or be configured using `parseImageUrl` callback, wich
should receive `image` or `plain_image` property and return an url string.

## 3 sections ranking

<Canvas>
    <Story name="3 sections" args={{ '.rankingData': rankingData }}>
        {Template.bind({})}
    </Story>
</Canvas>

## 3 sections ranking as grid

A particular behaviour of our ranking is the ability to show multiple articles per columns. 
As you can see in this example you can configure it to show two columns inside each columns, so users can see more articles.

```html
    <glyph-ranking data="data" inner-columns="2" rows="2" inner-gap="var(--gui-padding--xs)"></glyph-ranking>
```

<Canvas>
    <Story name="3 sections as grid" args={{ '.rankingData': rankingData, '.innerColumns': 2, rows: 2, '.innerGap': 'var(--gui-padding--xs)' }}>
        {Template.bind({})}
    </Story>
</Canvas>

## Single section ranking

When we are going to render just one section of data, we can show it as `columns` or as grid (see next section).

<Canvas>
    <Story name="Single section" args={{ '.rankingData': sectionData }}>
        {Template.bind({})}
    </Story>
</Canvas>

## Single section ranking as grid

Single section can be shown as a `grid` so users can see more articles at the same time. To do this you'll have to configure it with
as many columns and rows as yout want.

```html
    <glyph-ranking data="data" columns="6" row="3" gap="var(--gui-padding--xs)"></glyph-ranking>
```

<Canvas>
    <Story name="Single section grid" args={{ '.rankingData': sectionData, columns: 6, rows: 3, gap: 'var(--gui-padding--xs)' }}>
        {Template.bind({})}
    </Story>
</Canvas>

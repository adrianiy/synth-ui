###### Configuration
# custom

Uses a custom middleware implemented on your side.

If it is configured with an object of properies we'll consider the first key the name of the middlewares and use the entire object as input argument.

> you can use middlewe as equivalent of custom: { middleware: ... }.
>
> ```yaml
> middleware: sortCountries
> ```
> ```yaml
> custom:
>   middleware: sortCountries
>   ```

#### Parameters

| NAME | TYPE | DESCRIPTION | DEFAULT VALUE|
|:-----|:-----:|:---------|------:|
|any|String or Object|name of customMiddleware or object with name as first key | null |

### Example 

```yaml
name: custom middleware
custom:
  middleware: sortCountries
```

```yaml
name: custom middlewares
custom:
  getMMTTdates:
    baseFiltersKey: filters
    filterKey: filters
    store: mmttFilters
```
